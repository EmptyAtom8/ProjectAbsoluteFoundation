List of Action and Features
Project Absolute Foundation – Backend & Full-Stack Learning Scope
Project Goal

Build a full-stack learning platform where the same standardized frontend functionality is implemented using different frontend frameworks (web and desktop), all backed by a single, well-defined backend API.

The focus is on:

understanding real-world backend responsibilities,

learning frontend–backend contracts,

practicing deployment and tooling across OSes,

and incrementally adding advanced features without over-engineering.

Backend Stack (Initial Phase)

Language: Python 3

Framework: Flask

Database: SQLite (initially)

Dependency Management: PDM

Execution: pdm run flask …

Environment: local development → LAN → VPS

The backend is designed to be framework-agnostic for the frontend, accessed purely through HTTP APIs.

Core Design Principles

API-first development

Frontend and backend communicate via a stable API contract.

Backend may change internally, but the contract remains stable.

Incremental complexity

Start with minimal endpoints.

Add features only when the frontend needs them.

Single source of truth

Backend owns validation, persistence, and audit logs.

Frontend focuses on UX, state, and interaction.

Review-driven learning

You implement features first.

Code is reviewed and improved iteratively.
